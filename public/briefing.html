6. RESPONSIVIDADE EXTREMA — 320px A 2560px NOVO v3

O site foi otimizado para funcionar em todas as resolucoes, de telas ultra-pequenas (320px) ate monitores ultrawide. Sistema de breakpoints customizado no Tailwind.

SISTEMA DE BREAKPOINTS

320pxCSS @media (max-width: 359px) overrides
xs: 360pxBreakpoint customizado Tailwind
sm: 640pxPadrao Tailwind
md: 768pxPadrao Tailwind
lg: 1024pxPadrao Tailwind
xl: 1280pxPadrao Tailwind

OVERRIDES 320px

  • Section padding: px-3 py-16
  • Watermark: clamp(60px, 20vw, 120px)
  • Botoes: px-5 py-3 text-base
  • Contadores: text-4xl (vs 5xl+)
  • Titulos: escala progressiva com xs:
  • Grids: 1 coluna em sub-360px

PAGINAS OTIMIZADAS PARA 320px

LANDING PAGE

  • Hero: texto 3xl xs:4xl
  • Navbar: logo xl xs:2xl
  • Problema: counters 3xl xs:5xl
  • Custo: labels w-16 xs:w-20
  • Calc: valores 2xl xs:3xl
  • FAQ: titulo 4xl xs:5xl
  • Galeria: titulos sm:2xl
  • CTA: titulo responsivo

CONFIGURADOR

  • Model grid: gap-2 xs:gap-3
  • Cards: p-2.5 xs:p-4
  • Imagem: h-24 xs:h-32
  • Titulos: text-base xs:text-xl
  • Cores: w-8 xs:w-10
  • CTA: px-4 xs:px-6

ADMIN PANEL

  • Sidebar: hamburger no mobile
  • Overlay backdrop com blur
  • Content padding: p-4 xs:p-6
  • Cards responsivos 1-2-3 cols
  • Modais full-width no mobile
  • Touch targets minimo 44px

TESTADO EM: Samsung S22 (360px), iPhone SE (375px), simulacao 320px. Todos os elementos acessiveis, sem overflow horizontal, sem texto cortado, touch targets de 44px minimo. O breakpoint xs: 360px permite ajustes finos sem quebrar telas maiores.
